# 日常事件追踪功能实施计划（Sidebar）

## 一、目标定义
- 在日历应用中新增「当日事件追踪」子模块，专注记录“今天”发生的所有行为事件（吃饭、上课、走路、刷手机、健身等）。
- 支持复盘与审计：每天可以看到时间花在哪里，并能回顾遗漏信息并补录。
- 功能入口放在 Sidebar，保持快速记录、低打扰、可持续使用。

## 二、需求拆解（对应 1~7）
1. 手动补录当日已完成事件（可填事件发生时间 + 持续时长）。
2. 根据当日日历事件自动入账（到点后自动生成“已发生记录”）。
3. 图表展示今日时间分配（同名事件合并统计，动态图更新）。
4. 输入事件时提供历史名称建议（自动补全/推荐）。
5. 支持“开始事件 → 稍后结束”，并允许结束时间转盘微调（开始时间精确，结束可修正）。
6. 支持“模糊记忆录入”（只记得大概时段与大概时长），系统给默认建议并可一键修正。
7. 支持删除事件。

## 三、分阶段实施计划

### Phase 1：数据模型与基础记录（先可用）
- 新增 `DailyActionEvent` 数据结构：
  - `id`
  - `name`
  - `source`（manual / auto_calendar / timer / fuzzy）
  - `startAt`
  - `endAt`
  - `durationMin`
  - `confidence`（exact / adjusted / fuzzy）
  - `createdAt`, `updatedAt`
- 约束：所有事件必须属于“当前自然日”。
- 完成：新增、编辑、删除、列表展示。

### Phase 2：自动化与可视化（形成闭环）
- 自动入账：监听当日日历事件，超过开始时间后自动生成记录（避免重复入账）。
- 图表：按事件名聚合 `durationMin`，输出饼图/条形图数据；事件变化后实时刷新。
- 名称建议：基于历史频率 + 最近使用时间进行排序推荐。

### Phase 3：计时器与容错录入（核心体验）
- 开始/结束模式：
  - 点击“开始”即写入精确 `startAt`（立刻持久化，防止页面关闭丢失）。
  - 点击“结束”默认当前时间，可用时间转盘微调 `endAt`。
- 模糊录入模式（重点）：提供“事件名 + 大致开始/结束 + 大致时长”的快速表单，自动给默认值并允许微调。

## 四、第六点重点方案设计（模糊记忆录入）

下面给你 3 套可落地方案，从“快上线”到“更智能”。

### 方案 A：规则映射型（最快上线，稳定）
- 思路：把“上午/中午/傍晚/吃完晚饭后/很快/一会/一下午”等描述映射到固定时间模板。
- 例子：
  - 上午 → 默认开始 09:30
  - 中午 → 默认开始 12:30
  - 吃完晚饭后 → 默认开始 19:30
  - 很快 → 15 分钟，一会 → 30 分钟，一个多小时 → 90 分钟，一下午 → 180 分钟
- 优点：实现快、行为可解释、调试简单。
- 缺点：个性化较弱，初期“猜你习惯”不够准。

### 方案 B：统计学习型（推荐作为主线）
- 思路：按“事件名”累计历史统计，计算：
  - 平均开始时间（可用分钟数均值）
  - 平均结束时间
  - 常见持续时长（均值 + 中位数）
  - 时间离散度（标准差或分位数区间）
- 录入时策略：
  - 若用户只输“吃饭 + 中午 + 一会”，系统先用“吃饭”历史统计修正，再用“中午/一会”语义作约束。
  - 输出一个默认候选（如 12:40~13:10）+ 两个备选（偏短、偏长）。
- 优点：越用越准，符合你的真实作息。
- 缺点：冷启动时数据少，需要默认模板兜底。

### 方案 C：混合置信度型（长期最优）
- 思路：A + B 融合，并给每条建议打分：
  - `score = 语义匹配分 + 历史统计分 + 最近7天习惯分`
- 行为：
  - 高分建议直接预填。
  - 中分建议给“推荐 + 备选”。
  - 低分建议只给粗粒度默认值（15/30/60/120/360）。
- 优点：兼顾可解释性、个性化和鲁棒性。
- 缺点：实现复杂度最高。

## 五、我建议的落地顺序
1. 先做方案 A（保证马上可用）。
2. 同步埋点和数据结构，为方案 B 准备统计数据。
3. 一周后切到方案 B 作为默认推荐逻辑。
4. 后续再迭代到方案 C（加入置信度与多候选推荐）。

## 六、默认值与词汇映射（首版建议）

### 时长默认桶
- 5 分钟、15 分钟、30 分钟、60 分钟、120 分钟、240 分钟

### 语义时段默认点
- 清晨 06:30
- 上午 10:00（周三/周四早八日可切换到 08:00）
- 中午 12:30
- 下午 15:00
- 傍晚 18:30
- 晚上 20:00
- 深夜 23:30

### 模糊词到时长
- 很快 = 5
- 一会 = 20
- 一阵 = 45
- 一个多小时 = 90
- 一下午 = 240

## 七、你当前习惯画像（已采集）
1. 三餐习惯：
   1. 早餐周中通常不吃；如果吃，多在 8 点前或 9~10 点。
2. “吃完晚饭后”：
   1. 通常在 19 点左右。
3. 模糊时长偏好：
   1. 一会 = 10~30 分钟，建议中心值 20 分钟。
   2. 很快 = 1~5 分钟，建议中心值 5 分钟。
   3. 一下午 = 240 分钟。
4. 作息规律：
   1. 周三周四早八会早起；其余日期通常 9:30 之后起床。
   2. 睡觉时间通常在 23:30。
5. 场景绑定：
   1. 健身等事件没有固定发生场景。
6. 建议呈现方式：
   1. 你偏好系统给 3 个候选值。

## 八、第六点的个性化落地策略（基于你的回答）
1. 候选值策略（默认三候选）：
   1. 候选 A（推荐）：历史统计中心值（中位数优先）。
   2. 候选 B（偏短）：P25 分位值（或中心值 - 20%）。
   3. 候选 C（偏长）：P75 分位值（或中心值 + 30%）。
2. 时间语义解析优先级：
   1. 明确输入（如“13:20”）> 场景词（如“吃完晚饭后”）> 习惯时段（上午/中午等）> 通用默认值。
3. “吃完晚饭后”特殊规则：
   1. 默认起点 19:00。
   2. 若当天已有“晚饭”记录，则默认起点 = 晚饭结束后 10 分钟。
4. 工作日微调规则：
   1. 周三周四上午相关事件默认候选整体前移 60~90 分钟。
   2. 其余日期上午事件默认候选以 10:00 附近为中心。
5. 冷启动策略（历史数据不足 5 条）：
   1. 使用本计划中的词汇映射 + 时间桶。
   2. 每次手动修正都写入“偏移学习”，用于下一次推荐纠偏。
6. 交互规则：
   1. 录入时默认展开 3 个候选卡片（开始、结束、时长联动）。
   2. 点选候选后仍可用转盘精调，不强制手输。

## 九、验收标准（MVP）
- 能在 Sidebar 完成新增/删除/编辑当日事件。
- 能开始/结束事件，且结束时间支持转盘调整。
- 能基于历史事件名给出输入建议。
- 能显示今日时间分配图并动态更新。
- 能使用模糊描述快速录入，并给出可编辑的默认时间建议。
